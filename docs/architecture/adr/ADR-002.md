### ADR-002: Use Docker Compose for Deployment

- **Status**: Accepted
- **Context**: The application needs a consistent deployment model for local development, testing, and demonstration that can also support the security and API behavior required by [QR-001](../../quality-requirements.md#qr-001-patch-gamesid-security-enforcement), [QR-002](../../quality-requirements.md#qr-002-delete-gamesid-security-enforcement), and [QR-003](../../quality-requirements.md#qr-003-public-author-api-contract).
- **Decision**: Use Docker Compose to orchestrate the frontend, backend, database, and observability services as a single reproducible stack.
- **Quality requirements addressed**: [QR-001](../../quality-requirements.md#qr-001-patch-gamesid-security-enforcement), [QR-002](../../quality-requirements.md#qr-002-delete-gamesid-security-enforcement), [QR-003](../../quality-requirements.md#qr-003-public-author-api-contract)
- **Consequences**: The setup is simple and portable, and the same runtime topology can be used across environments. The main limitation is that the deployment remains single-host and therefore has limited scalability and redundancy.
