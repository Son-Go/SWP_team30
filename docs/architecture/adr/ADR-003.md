### ADR-003: Use Nginx as Reverse Proxy

- **Status**: Accepted
- **Context**: The frontend and backend need to communicate through a single entry point while preserving a stable public interface for [QR-003](../../quality-requirements.md#qr-003-public-author-api-contract) and supporting the authenticated traffic patterns behind [QR-001](../../quality-requirements.md#qr-001-patch-gamesid-security-enforcement) and [QR-002](../../quality-requirements.md#qr-002-delete-gamesid-security-enforcement).
- **Decision**: Use Nginx to serve the React SPA and reverse-proxy `/api/` requests to the backend through one controlled gateway.
- **Quality requirements addressed**: [QR-001](../../quality-requirements.md#qr-001-patch-gamesid-security-enforcement), [QR-002](../../quality-requirements.md#qr-002-delete-gamesid-security-enforcement), [QR-003](../../quality-requirements.md#qr-003-public-author-api-contract)
- **Consequences**: The architecture gains a single entry point, simplified CORS handling, and static file serving. The trade-off is that gateway-level concerns such as routing and TLS termination must be maintained centrally.
