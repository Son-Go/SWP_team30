### ADR-004: Use Flyway for Database Migrations

- **Status**: Accepted
- **Context**: The database schema needs to be version-controlled and reproducible across environments so that the API contract and security-related data model remain dependable for [QR-003](../../quality-requirements.md#qr-003-public-author-api-contract) and the authenticated write operations in [QR-001](../../quality-requirements.md#qr-001-patch-gamesid-security-enforcement) and [QR-002](../../quality-requirements.md#qr-002-delete-gamesid-security-enforcement).
- **Decision**: Use Flyway for database migrations, with migration scripts stored in `classpath:db/migration` and executed automatically during application startup.
- **Quality requirements addressed**: [QR-001](../../quality-requirements.md#qr-001-patch-gamesid-security-enforcement), [QR-002](../../quality-requirements.md#qr-002-delete-gamesid-security-enforcement), [QR-003](../../quality-requirements.md#qr-003-public-author-api-contract)
- **Consequences**: Schema changes are version-controlled, repeatable, and auditable. The main drawback is that migration planning must be careful to avoid breaking existing data or API behavior.
