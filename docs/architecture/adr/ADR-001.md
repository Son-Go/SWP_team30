### ADR-001: Use JWT for Authentication

- **Status**: Accepted
- **Context**: The application needs stateless authentication for REST API endpoints while satisfying [QR-001](../../quality-requirements.md#qr-001-patch-gamesid-security-enforcement) and [QR-002](../../quality-requirements.md#qr-002-delete-gamesid-security-enforcement).
- **Decision**: Use JWT (JSON Web Tokens) with Spring Security for authentication and require bearer tokens on protected API operations. No server-side sessions are maintained.
- **Quality requirements addressed**: [QR-001](../../quality-requirements.md#qr-001-patch-gamesid-security-enforcement), [QR-002](../../quality-requirements.md#qr-002-delete-gamesid-security-enforcement)
- **Consequences**: Stateless authentication simplifies scaling and keeps the API horizontally scalable. The trade-off is that token storage, expiry handling, and revocation strategy must be managed carefully.
