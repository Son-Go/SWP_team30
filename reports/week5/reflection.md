# Reflection on Sprint-1

## Learning points

- Testing: Team must udpdate tests for new features as soon as possible, so it won't interfere with development
- Customer review: Team should schedule customer meeting long before actual meeting, so everyone can express their opinion on convenient time

## Validated assumptions
- **Confirmed:** customer changed priority of sprint-4 from dedicated user and admin profile pages to polishing UI of /games page and drafting profiles
- **Rejected:** we planned to stop development of /games page, but customer requested several more features for that page
- **Confirmed:** customer aproved visual style changes

## Friction and gaps
- Poorly configured testing resulted in blocking workflow on non-main branches
- Team discovered some misscomunication patterns which lead to misscomunications in API request bodies
- Due to unclear assignment descriptions and constantly changing customer expectations team morale went down
  
## Planned response

- Team reconfigured test, so they work only on main branch
- Team created dedicated document to display API requests changes
- Several team members decided to celebrate current project state by ordering sushi
