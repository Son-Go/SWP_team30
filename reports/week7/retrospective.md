# Sprint 5 Retrospective

**Date:** 19.07.2026  
**Participants:** @Son-Go, @the-shtorm, @venimu, @Zhend0sss, @grishinegor44-creator

## What went well

1. **Project transition:** Well architectured project allowed us to use simplified transition mechanism (just transfer git repo and write docker deployment documentation)
2. **Admin functionality:** Team added comprehensive admin panel, so admin can see users, ban them, see games, edit or ban games
3. **Documentation preparation:** The team updated user facing documentaion so now it up to date and fully prepared for project transition

## What did not go well

1. **Project transition:** Due to inability to properly communicate with customer at this week (customer was affected by internet shortage, because he was in another region), we wasnt able to conduct proper transition meeting
2. **Task traceability:** Not every implemented Sprint 5 task had a dedicated GitHub issue at the beginning of the sprint. This makes it harder to link roadmap items, Pull Requests, testing evidence, and completed work.
3. **Task separation:** due to vague project priorities, entire last week was entirely about frontend changes (frontent interface for already implemented admin and user profile endpoints and "about us" page), team was forced to move backend developers to frontend to succeed with sprint

## Action points

1. Create or update GitHub issues for all Sprint 5 moderation, authorization, testing, and documentation tasks, then link them to the relevant milestone and Pull Requests.
2. find customer after sprint and complete transition with proper meetings and discussions
3. Next time our team will be aware about our limitied powers during sprint and plan project in advance, to avoid such "all hands to frontend" situation
