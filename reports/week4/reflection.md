# Reflection on Sprint-1

## Learning points
- Testing: Testing must be incorporated into workflow in early stages of development. Test integration during final days of sprint led to incomplete test coverage during deployment
- Customer review: Presenting a live demo rather than screenshots led to more specific and actionable feedback

## Validated assumptions
- **Confirmed:** customer changed priority of sprint-3 from event parsing to refinign /game page and adding special subsections to the page
- **Rejected:** we planned to implement event parsing during sprint 3, but customer decided to remove this feature from project scope
- **Confirmed:** customer aproved visual style changes

## Friction and gaps
- Due to inefficient work planning, most of whis week code was created and merged before the tests appeared
- Due to lack of knowledge and process complexity, team abandoned proper implementation of several security functions, as they were not important in terms of current project state
- Team morale went down due to unclear assignment descriptions and constantly changing customer expectations
  
## Planned response

- Team will adjust workflow, to merge all code at least into main branch only when all necessary tests are ready
- Several team members decided to celebrate current project state by atennding the bar 
- Team will return to abandoned security functions in dedicated debug sprint
