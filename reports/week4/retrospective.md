# Sprint 1 Retrospective

**Date:** 27.06.2026
**Participants:** @Son-Go, @the-shtorm, @venimu, @Zhend0sss, @grishinegor44-creator

## What went well

1. **Role splitting:** team naturally splitted on frontend, backend and devops part, where each team member completed his own tasks
2. **Sprint planning:** team efficiently identified arguable tasks and implemented only important to customer functionality. Vaguely described tasks with arguable business value (such as event parsing) were abandonned (at sprint review customer finally desided to postpon such functionality because desided, that this functionality is useless)
3. **Future sprint planning:** team prepared meaningfull questions for sprint review, so customer was able to clearly describe, which result we should produce during next spint

## What did not go well

1. **Test implementation** due to inefficient work planning, most of whis week code was created and merged before the tests appeared
2. **Development process:** Due to lack of knowledge and process complexity, team abandoned proper implementation of several security functions, as they were not important in terms of current project state
3. **Team spririt:** Due to unclear assignment descriptions and constantly changing customer expectations team morale went down

## Action points

1. Team will return to abandoned security functions in dedicated debug sprint
2. Team will adjust workflow, to merge all code at least into main branch only when all necessary tests are ready
3. Several team members decided to celebrate current project state by atennding the bar
